# PRD — PaperMind

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | PaperMind |
| 한줄 설명 | PDF 논문 및 보고서를 자동 분석해서 구글 스프레드시트로 정리해주는 로컬 앱 |
| 유형 | 로컬 웹앱 (추후 배포 예정) |
| 난이도 | 초급~중급 |

---

## 2. 사용자 시나리오

**누가:** 논문 및 보고서를 많이 다루는 연구자 및 대학원생

**시나리오 1 — 기존 파일 정리**
컴퓨터에 흩어져 있던 논문 및 보고서 284개(497MB)를 폴더 하나에 모아두고, 앱을 실행해서 한 번에 자동 분석 후 구글 스프레드시트에 정리한다.

**시나리오 2 — 새 파일 추가**
새로 다운로드한 논문 또는 보고서를 폴더에 넣고 앱에서 버튼 하나로 추가 처리한다. 이미 정리된 파일은 건너뛰고 새 파일만 처리된다.

---

## 3. 핵심 기능 목록

### 필수 기능
- PDF 폴더 지정 및 전체 스캔
- 파일 유형 자동 감지 (논문 / 보고서 구분, 파일명이 아닌 내용 기반)
- 중복 파일 자동 감지 및 제거 (파일 해시 + 제목 이중 확인)
- Claude API를 활용한 정보 자동 추출 (결과물은 한국어로 출력)
- 구글 스프레드시트 자동 업데이트 (4개 시트)
- 배치 처리 진행상황 실시간 표시
- 실패 파일 로그 기록 (스캔 PDF 포함)
- 새 파일 추가 모드 (기처리 파일 건너뜀)
- 500페이지 이상 대용량 PDF 챕터별 분할 처리

### 선택 기능 (추후)
- 논문/보고서 간 키워드 기반 연관관계 시각화 (지식그래프)
- 모바일 접근을 위한 배포

---

## 4. 기술 스택

| 역할 | 기술 |
|------|------|
| UI + 로직 | Python + Streamlit |
| PDF 파싱 | PyMuPDF |
| AI 분석 | Anthropic Claude API (claude-sonnet-4-6) |
| 스프레드시트 | Google Sheets API + 서비스 계정 |
| 환경변수 관리 | python-dotenv (.env 파일) |
| 배포 (추후) | GitHub + Streamlit Cloud |

---

## 5. 화면 구성

### 화면 1 — 메인 (설정)
- PDF 폴더 경로 입력
- 구글 스프레드시트 URL 입력
- 실행 버튼 2개: "전체 배치 처리" / "새 파일만 추가"

### 화면 2 — 처리 중
- 진행 상황 표시 (예: 23/284 처리 중...)
- 현재 처리 중인 파일명 표시
- 성공/실패 카운트 실시간 표시

### 화면 3 — 완료
- 처리 결과 요약 (성공 N개, 실패 N개)
- 실패한 파일 목록 및 이유
- 스프레드시트 바로가기 링크

## 6. 상세 기능 명세

### 논문 추출 항목
- APA 인용 형식 (자동 생성)
- 제목, 저자, 출판연도, 저널명
- 키워드
- 연구목적 (한국어)
- 연구방법 (한국어)
- 연구결과 (한국어)

### 보고서 추출 항목
- 인용 형식 (자동 생성)
- 제목, 발행기관, 발행연도
- 키워드
- 배경 (한국어)
- 주요내용 (한국어)
- 시사점 (한국어)

### 구글 스프레드시트 구조

**시트 1 "논문 목록"** — 논문 빠른 검색용

| 컬럼 | 설명 |
|------|------|
| # | 논문 번호 |
| APA 인용 | APA 스타일 전체 인용 |
| 제목 | 논문 제목 |
| 저자 | 저자명 |
| 연도 | 출판연도 |
| 저널 | 저널/학회명 |
| 키워드 | 논문 키워드 |

**시트 2 "논문 상세요약"** — 논문 깊은 내용

| 컬럼 | 설명 |
|------|------|
| # | 논문 번호 (시트1과 연동) |
| 제목 | 논문 제목 |
| 연구목적 | 이 연구가 왜 수행되었는가 (한국어) |
| 연구방법 | 어떤 방법으로 연구했는가 (한국어) |
| 연구결과 | 무엇을 발견했는가 (한국어) |

**시트 3 "보고서 목록"** — 보고서 빠른 검색용

| 컬럼 | 설명 |
|------|------|
| # | 보고서 번호 |
| 인용 | 인용 형식 |
| 제목 | 보고서 제목 |
| 발행기관 | 국가기관 또는 기업명 |
| 연도 | 발행연도 |
| 키워드 | 보고서 키워드 |

**시트 4 "보고서 상세요약"** — 보고서 깊은 내용

| 컬럼 | 설명 |
|------|------|
| # | 보고서 번호 (시트3과 연동) |
| 제목 | 보고서 제목 |
| 배경 | 왜 이 보고서가 작성되었는가 (한국어) |
| 주요내용 | 핵심 내용 요약 (한국어) |
| 시사점 | 주요 시사점 및 제언 (한국어) |

### 중복 처리 로직
1. 파일 해시로 동일 파일 먼저 제거
2. 처리 후 제목 기준으로 스프레드시트 중복 재확인
3. 중복 감지 시 건너뛰고 로그 기록

### 대용량 PDF 처리
- 100페이지 이상 시 챕터별 분할 처리
- 분할 결과 합산해서 하나의 행으로 정리

## 7. 디자인 가이드
- 심플하고 깔끔한 학술적 UI
- 불필요한 장식 없이 기능 중심
- 진행상황은 프로그레스 바로 시각화
- 색상: 화이트 배경, 다크 텍스트, 포인트 컬러 1가지

## 8. 제약사항
- 인터넷 연결 필수 (Claude API, Google Sheets API)
- Anthropic API 키 및 구글 서비스 계정 JSON 파일 사전 발급 필요
- 스캔된 이미지 PDF는 텍스트 추출 불가 → 실패 로그에 기록 후 사용자가 직접 처리
- API 비용 발생 (284개 파일 기준 약 $3~5, 월말 카드 자동결제)
